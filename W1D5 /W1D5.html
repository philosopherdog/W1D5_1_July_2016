Selectors:

  • A name used to refer to a method in order to execute it or pass it around

  • Simply identifies a method.

  • The selector of a method is just the method name minus return type, and parameters.

- (void)testSelectors {
// notice the colon
SEL mySelector = @selector(myMethodWithData:);
[self performSelectorOnMainThread:mySelector withObject:[NSData new] waitUntilDone:YES];
SEL myOtherSelector = @selector(myMethodWithString:value:);
}
- (void)myMethodWithData:(NSString *)data {}
- (NSData *)myMethodWithString:(NSString *)string value:(BOOL)value {
return [NSData new];
}
  • 2 common ways to get a selector
- (void)testSelectors {
// compile time
SEL aSelector1 = @selector(fly);
// run time
SEL aSelector2 = NSSelectorFromString(@"nameOfMethod");
}

  • Many framework methods expect a selector as a parameter.

  • For instance, if we want to programmatically setup a target-action on a button (we'll be discussing UIButton next week) we will call the instance method:

/*
// definition
- (void)addTarget:(id)target action:(SEL)action forControlEvents:(UIControlEvents)controlEvents;
*/
- (void)testButtonSelectorArgument {
// adding it to a button
UIButton *aButton = [[UIButton alloc] initWithFrame:CGRectZero];
[aButton addTarget:self action:@selector(buttonTapped:) forControlEvents:UIControlEventTouchUpInside];
}
// actual method the button calls when tapped
- (void)buttonTapped:(UIButton *)sender {
// do stuff
}
  • A common use of selectors is to test whether an object can handle a message.
// Protocol
@protocol MyProtocol <NSObject>
@optional
- (void)someMethod;
@end
// Class
@interface MyObj : NSObject<MyProtocol>
@end
@implementation MyObj
@end
- (void)testSelector {
MyObj *myObj = [MyObj new];
if ([myObj respondsToSelector:@selector(someMethod)]) {
[myObj performSelector:@selector(someMethod)];
// or, since it responds, do this:
// [myObj someMethod];
}
}
// you can perform a selector after a delay
- (void)testPerformSelectorAfterDelay {
[self performSelector:@selector(myMethod) withObject:nil afterDelay:1.0];
}
- (void)myMethod {
NSLog(@"works!");
}
  • This is a handy way of sorting an array using a selector.
- (void)testArraySort {
NSArray *unsorted = @[@"Hello", @"Light", @"House", @"Labs"];
NSArray *sorted = [unsorted sortedArrayUsingSelector:@selector(compare:)];
NSArray *expected = @[@"Hello", @"House", @"Labs", @"Light"];
XCTAssert([sorted isEqualToArray:expected]);
}

More Protocols & Delegation

What are protocols?

  • In the real world protocols consist of sets of agreed upon procedures, rules or conventions for doing stuff.

  • E.g. police follow a legally binding protocol when making an arrest.

  • They read you your rights in a specific format, etc.

  • Computers communicate on the internet using the http protocol.

  • The http protocol defines the expected request and the expected response data and format.

  • There would be no internet without a shared protocol.

  • In iOS a protocol usually consists of a group of method signatures (and sometimes properties) that any conforming class agrees to implement.

  • Protocol methods can be optional or required.

  • Required methods must be implemented.

  • Optional methods need not be implemented. So, we always need to check whether an optional protocol method is implemented before sending the message.

  • Protocols are similar to interfaces in other languages.

Why are protocols important?

  • Protocols are used everywhere in Cocoa and CocoaTouch especially as part of the delegate design pattern.

  • If some class agrees to implement a protocol, then other objects can communicate with this object without needing to know any other details about the object. This is a good example of loose coupling. Why is "loose coupling" a good OO design principle?

  • Identifying objects just by their conformance to a protocol is a big deal in many design patterns.

Protocol Syntax

// Protocols can inherit from other protocols
@protocol MyProtocol<NSObject>
- (void)putYourMethodsHere;
@end
// Optional/required
@protocol AnotherProtocol<MyProtocol>
// @required is default
- (void)putYourMethodsHere;
// optional
@optional
- (void)optionalMethod;
// use @required to switch back
@required
- (NSString*)requiredAgain;
@end
// Conformance syntax
@interface MyClass:NSObject<AnotherProtocol>
// don't put the signatures in the header
@end
@implementation MyClass
// required
- (NSString*)requiredAgain {
return @"Some result";
}
// required
- (void)putYourMethodsHere {
// do stuff
}
@end
// Testing protocol conformance
- (void)testProtocol {
BOOL conforms = [MyClass conformsToProtocol:@protocol(AnotherProtocol)];
XCTAssertTrue(conforms);
MyClass *myClass = [MyClass new];
BOOL responds = [myClass respondsToSelector:@selector(optionalMethod)];
XCTAssertFalse(responds);
}

Example Of Protocols & Polymorphism

// Flyable.h
@protocol Flyable <NSObject>
- (NSString *)fly;
@end
// Duck.h
//#import "Flyable.h"
@interface Duck : NSObject<Flyable>
@end
// Duck.m
//#import "Duck.h"
@implementation Duck
- (NSString *)fly {
return @"flyin high!";
}
@end
// RubberDuck.h
//#import "Flyable.h"
@interface RubberDuck : NSObject<Flyable>
@end
// RebberDuck.m
//#import "RubberDuck.h"
@implementation RubberDuck
- (NSString *)fly {
return @"can't fly worth beans";
}
@end
- (NSString *)executeFlyableObject:(id<Flyable>)aFlyable {
return [aFlyable fly];
}
- (void)testDucks {
id<Flyable>bird1 = [Duck new];
id<Flyable>bird2 = [RubberDuck new];
NSArray *arr = @[bird1, bird2];
for (id<Flyable>item in arr) {
[item fly];
}
NSString *result1 = [self executeFlyableObject:bird2]; // ==> can't fly worth beans
XCTAssert([result1 isEqualToString:@"can't fly worth beans"]);
NSString *result2 = [self executeFlyableObject:bird1]; // ==> flyin high!
XCTAssert([result2 isEqualToString:@"flyin high!"]);
}

Simple Delegation Example

// Basic Delegation Example Showing How To Get Another Object To Do Work For A Class
// This allows
#import <Foundation/Foundation.h>
// Protocol
@protocol PlayerDelegate <NSObject>
- (void)play;
@end
// Apple Service
@interface AppleMusicService : NSObject<PlayerDelegate>
@end
@implementation AppleMusicService
- (void)play {
NSLog(@"playing apple music playlist");
}
@end
// Spotify Service
@interface SpotifyService : NSObject<PlayerDelegate>
@end
@implementation SpotifyService
- (void)play {
NSLog(@"playing spotify playlist");
}
@end
// Player
@interface Player : NSObject
@property (nonatomic, weak) id<PlayerDelegate>delegate;
- (instancetype)initWithMusicService:(id<PlayerDelegate>)service;
- (void)play;
- (void)changeServiceTo:(id<PlayerDelegate>)service;
@end
@implementation Player
- (instancetype)initWithMusicService:(id<PlayerDelegate>)service {
if (self = [super init]) {
_delegate = service;
}
return self;
}
- (instancetype)init {
NSAssert(NO, @"Use designated initializer instead");
return nil;
}
- (void)play {
[self.delegate play];
}
- (void)changeServiceTo:(id<PlayerDelegate>)service {
if ([service isMemberOfClass:[self.delegate class]]) {
return;
}
self.delegate = service;
}
@end
- (void)testPlayer {
AppleMusicService *appleMusic = [AppleMusicService new];
SpotifyService *spotify = [SpotifyService new];
Player *player = [[Player alloc] initWithMusicService:appleMusic];
[player play];
[player changeServiceTo:spotify];
[player play];
}

Simple Delegate Callback Example

// Detail.h
@class Detail;
@protocol DetailDelegate <NSObject>
- (void)detail:(Detail *)detail doStuffWithData:(NSString *)data;
@end
@interface Detail : NSObject
@property (nonatomic, weak) id<DetailDelegate>delegate;
- (void)saveFakeUserInput:(NSString *)input;
@end
// Detail.m
//#import "Detail.h"
@implementation Detail
- (void)saveFakeUserInput:(NSString *)input {
[self.delegate detail:self doStuffWithData:input];
}
@end
// Master.h
@interface Master : NSObject
- (void)fakeButtonTap;
@property (nonatomic, strong) Detail *detail;
@end
// Master.m
//#import "Master.h"
// class extension, not really used execept for conforming to DetailDelegate
@interface Master()<DetailDelegate>
@end
@implementation Master
- (void)fakeButtonTap {
self.detail.delegate = self;
// you might want to segue to detail
}
- (void)detail:(Detail *)detail doStuffWithData:(NSString *)data {
NSLog(@"%s data: %@", __PRETTY_FUNCTION__, data);
}
@end
- (void)test {
Master *m = [Master new];
Detail *d = [Detail new];
m.detail = d;
[m fakeButtonTap];
[d saveFakeUserInput:@"some user input"];
}

NSNumber

  • Light weight wrapper around primitive integer types.
  • Most often used to include number values in collections in Objective-C.
  • For instance, to include integers in an NSArray convert to NSNumber
// 3 different ways to instantiate
// prefer literal instantiation
- (void)test {
NSNumber *num1 = [[NSNumber alloc] initWithInt:22];
NSNumber *num2 = [NSNumber numberWithFloat:12.2];
NSNumber *num3 = @(33);
NSNumber *num4 = @(YES); // BOOL
NSNumber *num5 = @('i'); // Char
NSArray *arr = @[num1, num2, num3, num4, num5];
}
  • You may need to unbox NSNumbers. Do it like this:
- (void)test {
NSInteger unwrappedNum1 = [arr[0] intValue];
NSLog(@"%lu", unwrappedNum1);
float unwrappedNum2 = [arr[1] floatValue];
NSLog(@"%f", unwrappedNum2);
NSInteger unwrappedNum3 = [arr[2] intValue];
NSLog(@"%lu", unwrappedNum3);
BOOL val = [arr[3] boolValue];
NSLog(@"%@", val ? @"YES": @"NO");
// char: What will these logs print?
NSLog(@"char value boxed %@", arr[4]); // prints unicode value
NSLog(@"char value unboxed: %c", [arr[4] charValue]); // prints character i
}
  • Some Tricks
- (void)test {
// using NSNumber's literal syntax as a dictionary key!
NSDictionary *dict = @{@1:@"One", @2:@"Two", @3:@"Three"};
// looping: dict.allKeys gets an array of keys, but notice it has no definite order
// dictionaries are unordered
for (NSNumber *key in dict.allKeys) {
NSLog(@"%@", dict[key]);
}
NSInteger num5 = 44;
// logging primitive integer types by wrapping them in an NSNumber literal syntax
NSLog(@"logging an NSInteger by wrapping it: %@", @(num5));
// this is a quick way to get the string value of an integer type
NSString *num5ToString = @(num5).stringValue;
// this is the long way of doing the same thing
num5ToString = [NSString stringWithFormat:@"%d", 44];
}
  • Comparing NSNumbers
// Question: What will the statement at line 7 log out and why?
- (void)test {
NSNumber *num7 = @(22);
NSNumber *num8 = [NSNumber numberWithInteger:22];
BOOL value2 = num7 == num8; // this is a pointer comparison, likely not what you want!
NSLog(@"%@ is equal to %@: %@", num7, num8, value2 ? @"YES" : @"NO");
// do comparisons like this for NSNumber
// unboxing to compare
if ([num7 intValue] == [num8 intValue]) {
NSLog(@"they're equal");
}
// comparing while boxed
if ([num7 isEqualToNumber:num8]) {
NSLog(@"they're equal yo");
}
}
// This is another way of comparing, just a FYI, since you may see similar "sentinels" used elsewhere
// Don't do this for NSNumber (it's just an illustration)
- (void)test {
NSNumber *num7 = @(22);
NSNumber *num8 = [NSNumber numberWithInteger:22];
NSComparisonResult comparisonResult = [num7 compare:num8];
if (comparisonResult == NSOrderedAscending) {
NSLog(@"ascending");
} else if (comparisonResult == NSOrderedSame) {
NSLog(@"same");
} else if (comparisonResult == NSOrderedDescending) {
NSLog(@"descending");
}
}

Objective-C Categories

What are categories:

  • Called Extensions in Swift.

  • Add functionality to existing classes without modifying original class.

  • Can modify private system classes (that you can't even see!) without subclassing.

  • Can be used to break up complex classes into logical components.

  • Allows flexibility of adding functionality as needed. For instance, I could add an extension to NSString but choose to only use it in some classes and not others. So, not every NSString in my project would automatically get the next behaviour (this isn't true in Swift)

File Naming Convention

NameOfExtendedClass+NameOfExtension.h/.m

e.g.
NSString+Utilities.h/.m

  • You need to import the category to get the functionality (in Objc).

Syntax

  • interface + implementation just like classes
  • But the syntax is a bit different than classes.
  • Notice the round brackets.
  • There's no superclass after a colon as in classes.
  • The name of the category is inside the round brackets after the name of the class being extended.
@interface NSString(Utils)
@end
@implementation NSString(Utils)
@end
  • Xcode will automatically create the files and stubs for you if you do this:

New File >> iOS Source >> Objective-C File >> Category



  • You refer to the object being acted on when you are inside the implementation as self.

e.g.

@interface NSString(Utils)
- (NSString *)addStar;
@end
@implementation NSString(Utils)
- (NSString *)addStar {
// notice SELF here to represent the NSString instance that receives this message
return [self stringByAppendingString:@"*"];
}
@end
- (void)test {
NSString *s = [@"steve" addStar];
XCTAssert([s isEqualToString:@"steve*"]);
}
// More advanced NSString Extension that returns the vowels on an NSString
// NSString+Vowels.h
@interface NSString (Vowelize)
- (NSString *)vowelize;
@end
// NSString+Vowels.m
#import "NSString+Vowels.h"
@implementation NSString (Vowelize)
- (NSString *)vowelize {
NSMutableString *result = [NSMutableString string];
if (self.length == 0) {
return [result copy];
}
NSString *comparitor = @"aeiou";
// loop through string
for (NSInteger i = 0; i < self.length; ++i) {
NSRange range = NSMakeRange(i, 1);
NSString *subStr = [self substringWithRange:range];
if ([comparitor localizedStandardContainsString:subStr]) {
[result appendString:subStr];
}
}
return [result copy];
}
@end
- (void)testVowelize {
NSString *vowels = [@"my vowel experiment" vowelize];
NSString *result = @"oeeeie";
XCTAssert([vowels isEqualToString:result]);
}

Objective C Class Extension

  • Way to add another interface to your classes that are not visible to outside classes.
  • They were more commonly used for methods in early versions of Objc where you had to forward declare all methods.
  • Modern Objc uses Class Extensions for properties only.
  • Always start by adding your properties to the class extension and only move them to the header if they need to be exposed. Why do I say this?
// Simple example of class extension
// Person.h
@import Foundation; // Notice the modern importation syntax
@interface Person: NSObject
// Notice age is readonly
@property (nonatomic, readonly) NSInteger age;
- (instancetype)initWithName:(NSString *)name age:(NSInteger)age;
@end
// Person.m
// #import "Person.h"
// class extension. notice it's another interface on .m
@interface Person()
// privately it's readwrite but publicly it's readonly
@property (nonatomic, readwrite) NSInteger age; // optional way of doing this, because you can write to age using _age privately
@property (nonatomic) NSString *name;
@end
@implementation Person
// this is called the designated initializer
- (instancetype)initWithName:(NSString *)name age:(NSInteger)age {
if (self = [super init]) {
_name = name;
_age = age;
}
return self;
}
// overriding the default init and calling the designated initializer and passes in defaults
- (instancetype)init {
return [self initWithName:nil age:0];
}
@end
- (void)test {
Person *p1 = [[Person alloc] init];
XCTAssert(p1.name == nil);
XCTAssert(p1.age == 0);
Person *p2 = [[Person alloc] initWithName:@"JJ" age:10];
XCTAssert(p2.name = @"JJ");
XCTAssert(p2.age == 10);
}
